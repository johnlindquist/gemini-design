Can you explain the difference between lists and tuples in Python?

Lists vs. Tuples

In Python, both lists and tuples are built-in data structures used to store collections of items, but they have one fundamental difference: mutability.

  • Lists are mutable: You can change, add, or remove elements after the list is created. They are defined using square brackets [].
  • Tuples are immutable: Once a tuple is created, its elements cannot be altered. They are defined using parentheses ().
python 
# This is a list (mutable)
my_list = ["apple", "banana", "cherry"]
my_list[0] = "orange"  # Valid: modifies the first element
my_list.append("grape")  # Valid: adds a new element

# This is a tuple (immutable)
my_tuple = ("apple", "banana", "cherry")
my_tuple[0] = "orange" # TypeError: 'tuple' object does not support item assignment

Because tuples are immutable, they are slightly more memory-efficient and generally faster to iterate over than lists.

When should I use each one?

Here is a comparison to help you decide when to use a list versus a tuple:

Feature List Tuple
Use Case Collections of items that may need to change during program execution (e.g., active user sessions, a shopping cart). Collections of items that should remain constant (e.g., configuration settings, coordinates, days of the week).
Syntax Square brackets [] Parentheses ()
Performance Slightly slower, consumes more memory. Faster iteration, more memory efficient.
Dictionary Keys Cannot be used as dictionary keys (unhashable). Can be used as dictionary keys (hashable).

Summary: Use a list if you expect the collection of data to grow, shrink, or change over time. Use a tuple when you want to ensure the data cannot be modified, which provides a guarantee of data integrity and a slight performance boost.
