Lists vs. Tuples in Python

Both lists and tuples are used to store collections of data in Python, but they have one fundamental difference: mutability.

  • Lists are mutable. You can add, remove, or change items after creation. They use square brackets [].
  • Tuples are immutable. Once created, their contents cannot be modified. They use parentheses ().
python 
# List example (Mutable)
my_list = [1, 2, 3]
my_list.append(4)
my_list[0] = 99
print(my_list)  # Output: [99, 2, 3, 4]

# Tuple example (Immutable)
my_tuple = (1, 2, 3)
# my_tuple.append(4)  # AttributeError
# my_tuple[0] = 99    # TypeError
print(my_tuple) # Output: (1, 2, 3)

AI

Choosing between a list and a tuple depends on how you intend to use the data.

Feature List Tuple
Mutability Mutable (can change) Immutable (fixed)
Syntax Square brackets [] Parentheses ()
Performance Slightly slower (dynamic memory) Faster (fixed memory allocation)
Use Case Collections of homogenous items that may change over time. Heterogeneous data or fixed collections (like coordinates).

As a general rule, use a list when you expect the collection to grow, shrink, or change. Use a tuple when you want to ensure the data remains constant throughout the program's execution, which also provides a slight performance benefit and allows it to be used as a dictionary key.
